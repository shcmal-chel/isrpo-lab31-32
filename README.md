## Краткая теория (вводная часть)

### Зачем нужна база данных?

В предыдущих лабораторных работах данные хранились в обычном списке в памяти. Это работало, но у такого подхода есть критический недостаток: при перезапуске сервера все данные пропадают.

### JSON и C#: соответствие типов

Способ хранения | Где живут данные | Что происходит при перезапуске
-|-|-
static List<T> в памяти | RAM (оперативная память) | Всё теряется
Файл (JSON, txt) | Диск |Сохраняется, но неудобно работать
База данных | Диск (с умнымдоступом) |Сохраняется + быстрый поиск/фильтрация

### Что такое SQLite?

**SQLite** — это встраиваемая реляционная база данных. В отличие от PostgreSQL или
MySQL, SQLite не требует отдельного сервера — вся база хранится в одном файле с расширением .db прямо в папке вашего проекта.

#### Почему SQLite идеально подходит для обучения:

Характеристика | SQLite
-|-
Установка | Не нужна — это просто файл
Сервер | Не нужен
Язык запросов | Стандартный SQL
Подходит для | Разработки, обучения, небольших проектов
Файл базы данных | имя.db в папке проекта

## Примеры кода:

```cs
[HttpDelete("completed")]
    public async Task<ActionResult> DeleteCompletedTasks()
    {
        var completedTasks = await _db.Tasks
            .Where(t => t.IsCompleted)
            .ToListAsync();
        _db.Tasks.RemoveRange(completedTasks);
        await _db.SaveChangesAsync();
        return Ok(new 
        { 
            Message = $"Удалено выполненных задач: {completedTasks.Count}",
            DeletedCount = completedTasks.Count 
        });
    }
```
